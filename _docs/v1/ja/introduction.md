# はじめに：魔法の図書館

**多言語ドキュメントサイト**へようこそ。このプロジェクトは単なる静的ウェブサイトではありません。最新のウェブアーキテクチャで構築された「魔法の図書館」です。

## 仕組み

### 1. Next.js (司書)

Next.jsは賢い司書です。誰かがあなたのウェブサイトを訪れると、ページ（「はじめに」など）を求めます。司書は走ってそれを取りに行き、見せてくれます。

- **React**: これらは司書がページを作るために使う**レゴブロック**です。あるブロックはボタン、別のブロックはサイドバーです。

### 2. Docker (魔法のお弁当箱)

通常、家で料理を作っても、友達の家では味が違うかもしれません。
**Docker**は**魔法のお弁当箱**のようなものです。キッチン全体（コード）を中に入れます。これで、どこでこのお弁当箱を開けても（あなたのコンピュータ、私のコンピュータ、またはインターネット）、ウェブサイトはまったく同じに見えます。

### 3. ISR (「更新」の魔法)

これは**増分静的再生成 (Incremental Static Regeneration)**、または**「コピー機」**を意味します。

- **ISRなし**: 司書は誰かが頼むたびに手でページを書かなければなりません。遅い！
- **ISRあり**: 司書はページの**コピー**を作り、それを全員に配ります。超高速です！
- **ひねり**: 60秒ごとに、テキストを変更すると、司書は*新しい*コピーを作ります。

## 実例

### Netflix ヘルプセンター

Netflixが新機能を追加すると、ヘルプページを更新します。タイプミス1つのためにサイト全体（何百万ページ）を再構築したくありません。彼らは**ISR**を使って、その1ページだけを即座に修正します。

### Stripe ドキュメント

これらの企業は30以上の言語でドキュメントを持っています。彼らは**i18n**を使用しているため、`stripe.com/docs`は英語を表示しますが、`stripe.com/ja/docs`は自動的に日本語を表示します。

## 主な機能

- **国際化**: 英語、スペイン語、フランス語、ドイツ語、日本語で利用可能。
- **全文検索**: Fuse.jsを搭載。
- **APIリファレンス**: Swagger UI統合。
